# archive 目录

本目录保存从第一篇完整工程中归档的内容。当前仓库目标已切换为第二篇论文实验工程，因此这些文件不再作为主流程入口。

- `notebooks/`：旧 notebook，保留参考但不作为运行入口。
- `figs/`：旧绘图输出或绘图缓存。
- `results/`：旧结果缓存。
- `legacy/`：早期归档脚本，例如航班数据抽样脚本。
- `compat_wrappers/`：上一轮为第一篇工程整理时新增的兼容包装目录。第二篇工程改用 `core/`、`baseline/`、`governance/` 结构，因此移入归档。

不确定是否仍有参考价值的文件均先归档，没有直接永久删除。
